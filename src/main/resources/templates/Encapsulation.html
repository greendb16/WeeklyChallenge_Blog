Encapsulation

Encapsulation is one of the fundamental concepts of Object Oriented Java. You may be familiar with it if you have ever used methods. Methods let you separate your data from how it is used. This can be useful for security and maintaining the integrity of your data. also lets use utilise the same data in many different places without having to create tons of identical variables, and ensuring that those variable maintain the same variables across classes and uses. Encapsulation is achieved by maintaining a set of private variables that the user does not have access to. Instead there arte a set of functions called Getters and Setters. These are used to instead set the variables and get those variables at the right time place. You can see an example of this setup below:




In this pic you can see that there are a number of "set..." methods. these can be used elsewhere to securely set those variables. Then there are the "get..." methods. You can see these in the toString() method used in the bottom of the picture recalling whatever the active value for that variable is. This allows you to keep your program upt o date despite changing variables
